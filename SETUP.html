Habit Calendar — Setup Guide

No GitHub needed. Just drag-and-drop. About 10 minutes total.

What's in the zip:
index.html — the app · functions/ — the API · 1-create-table.sql + 2-seed-data.sql — pasted into D1 · SETUP.html — this file
1
Create D1 database & load your data Free

D1 is Cloudflare's free SQL database. It will hold all your habit data plus the 144 reading-days already filled in for you.

↗ Open D1 in Cloudflare Dashboard

Steps:

1. Click "Create database"

2. Name it exactly: habit-calendar

3. Click "Create" → open the database → click the "Console" tab

4. Two separate pastes (the D1 console can't handle both at once):

→ Open 1-create-table.sql in a text editor. Copy all of it, paste into the D1 Console, click "Execute". You should see a success message.

→ Clear the console. Open 2-seed-data.sql. Copy all of it, paste, click "Execute". You'll see "144 rows affected" or similar.

What this does: creates the habits table AND inserts 144 days (Jan 1 → May 24, 2026) with the "Read 20 pages" box pre-checked. Days will glow orange because only 1 of 4 items is done — you can come back and add pushups / notes anytime to upgrade the glow.

If you tried the old single-file paste and got an error: safe to re-run — INSERT OR IGNORE skips dates that are already there.
2
Create Pages project & upload files

Cloudflare Pages can host your app without any GitHub setup — just upload files directly.

↗ Open Workers & Pages

Steps:

1. Click "Create" → tab "Pages" → "Upload assets"

2. Project name: habit-calendar (or whatever you want — this becomes your URL)

3. Drag the entire unzipped folder onto the upload area. The structure must look like this when Cloudflare sees it:

├── index.html ← required at root ├── SETUP.html ← optional, harmless ├── 1-create-table.sql ← optional, harmless ├── 2-seed-data.sql ← optional, harmless └── functions/ └── api/ ├── auth.js └── habits.js

4. Click "Deploy site" — wait ~30 seconds.

If drag-drop won't work: use the "Browse" button and select the contents of the folder (not the folder itself). The files must be at the upload root, not nested inside another folder.
3
Bind D1 to your Pages project

Open your new project → Settings → Bindings (under "Functions" section in some accounts).

Add a D1 database binding:

Variable name
DB
D1 database
habit-calendar

Click "Save".

Important: Variable name must be exactly DB — uppercase, the code references it by this name.
4
Set your admin password

Still in Settings → scroll to "Variables and Secrets".

Click "Add" and fill in:

Variable name
ADMIN_PASSWORD
Value
your-secret-password

Set type to "Secret" so it's hidden. Apply to both Production and Preview.

This is what unlocks editing on any device. Without it, visitors can only view your calendar — they can't check boxes, edit notes, or save anything.
5
Redeploy to activate bindings

The D1 binding and password only take effect on the next deployment, so trigger one:

Go to Deployments tab → find your latest deployment → click the ··· menu → "Retry deployment".

That's it. Open your URL (something like habit-calendar.pages.dev) — you'll see 144 orange-glowing days waiting for you. Click the 🔒 Admin button, enter your password, and you can start checking the other boxes.

✅ You're live!

To update the app later, go to Pages → Deployments → "Create new deployment" and drag the folder again.
